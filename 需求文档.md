## 一、项目概述

GitWeave 是一个面向机器人研发团队（UMI/REBOT/EGO/AlohaMini）的内部管理中枢，将代码版本控制、项目任务流转、问题追踪、人员权限管理和 Agent 使用经验分享深度整合。核心设计理念是"透明化研发脉络"——让管理者能一眼看到谁负责哪些任务、代码变更历史、以及任务移交轨迹。

## 三、功能模块

### 3.1 仪表盘概览
- Sprint 倒计时、累计提交数、活跃分支数（滚动数字动效）
- 7个项目卡片：进度条、任务数、待修复数、成员头像
- 任务分布条：按项目展示待办/进行中/已完成
- 移交轨迹速览

### 3.2 项目维度（7个项目）
| 项目 | 负责人 | 成员 |
|------|--------|------|
| UMI 系列 | 陈润峰 | 陈润峰、薛炫豪 |
| 触觉手套 | 王一帆 | 王一帆 |
| 传感器算法 | 薛炫豪 | 薛炫豪、陈露佳 |
| 机械臂训练 | 肖文博 | 肖文博、张柯 |

| 产品交付 | 刘世文 | 刘世文 |

### 3.3 项目看板（Kanban）
- 三列：待办 / 进行中 / 已完成
- 拖拽切换状态
- **项目筛选**：可按项目查看任务
- **人员筛选**：点击成员头像，只看该成员负责的任务
- **权限标识**：每张卡片显示"可编辑"或"只读"状态标签
- **移交功能**：卡片右上角「移交」按钮，选择新负责人+填写原因
- **移交记录**：卡片底部小头像链展示移交历史

### 3.4 权限系统（核心）
- 每个成员只能编辑自己被 assign 的任务
- **移交后权限变更**：
  - 原负责人（移交者）→ 只能查看，不能编辑
  - 新负责人（被移交者）→ 获得编辑权限
  - Admin → 始终全权限
- 任务卡片上显示权限状态标签（可编辑/Admin/只读）

### 3.5 移交轨迹追踪（Git Graph 风格）
- 横向布局：8位成员的纵向"轨道"
- 彩色曲线连接任务/问题在不同成员之间的移交
- 颜色编码：蓝=UMI，绿=触觉，红=传感器，黄=机械臂，紫=EGO，青=AlohaMini，橙=产品交付
- 悬停弹出详情：项目名称、任务标题、移交双方、时间
- 项目筛选

### 3.6 代码版本控制（核心）
- **权限规则**：
  - 普通成员：只能上传，不能下载/编辑
  - Admin：可以下载和编辑
  - 上传后自动递增版本号（v1 → v2 → v3，不覆盖）
- **版本历史**：
  - 每个版本有 Git Graph 演进图
  - 每个版本有 Diff 描述（+添加了什么 / -移除了什么）
  - 时间线展示：版本号、描述、上传者、时间、文件大小、hash
- 项目筛选

### 3.7 问题追踪
- 项目关联 + 项目筛选
- 状态筛选：待处理 / 已解决 / 已关闭
- 展开详情：问题描述 + 解决方案
- **移交时间线**：完整记录问题分配变更历史

### 3.8 Agent 使用心得
**不是项目内容**，而是 Agent 框架/工具的使用经验分享：
- 什么框架好用（ohMyPi vs Claude Code CLI 等）
- 什么 LLM 理解力强（GPT-4 vs Claude-3.5-Sonnet 等）
- 本地部署方案（Cursor + Ollama + CodeLlama）
- Prompt Engineering 技巧
- 框架选型指南和最佳实践
- 支持：发布心得、点赞、评论、标签筛选、搜索

### 3.9 Admin 管理控制台
- 添加入职成员（姓名、职位、缩写、颜色）
- 删除/标记离职成员
- 在职/离职人员统计
- Admin 和普通成员分区展示

## 四、视觉设计

- **深色开发者主题**：#050507 极深背景 + 等距网格线动画
- **渐变强调色**：红(#d7244b) 蓝(#1868d6) 绿(#10b981) 对应不同状态
- **玻璃态面板**：毛玻璃效果 + 细腻边框
- **JetBrains Mono** 等宽字体，契合代码风格
- **Three.js 3D 背景**：拓扑连线 + 滚动形变效果
